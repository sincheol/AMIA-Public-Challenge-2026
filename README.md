# AMIA Public Challenge 2026: Chest X-ray Abnormalities Detection

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 **AMIA Public Challenge 2026** 흉부 X-ray 객체 탐지 대회에 참가하여 수행한 모델링 및 최적화 코드를 담고 있습니다. 
흉부 방사선 사진에서 14가지 유형의 흉부 이상(Abnormalities)을 자동으로 위치 추적하고 분류하는 것이 목표입니다. 데이터셋은 숙련된 영상의학과 전문의가 주석을 단 15,000장의 스캔(학습용 8,573장, 테스트용 6,427장)으로 구성되어 있습니다.

- **평가 지표 (Evaluation Metric)**: PASCAL VOC 2010 mean Average Precision (mAP) at IoU > 0.4
- **제출 형식 (Submission)**: 이미지당 예측된 객체의 클래스 ID, 신뢰도(Confidence), 바운딩 박스 좌표(xmin, ymin, xmax, ymax)를 제출하며, 병변이 없는 정상 이미지의 경우 `14 1.0 0 0 1 1`로 제출해야 합니다.

## 2. 데이터 분석 및 주요 문제 정의
탐색적 데이터 분석(EDA)을 통해 도출한 데이터셋의 4가지 주요 특성은 다음과 같습니다.
1. **클래스 불균형 (Class Imbalance)**: 기흉(Pneumothorax) 등 소수 클래스와 대동맥 확장(Aortic enlargement) 등 다수 클래스 간의 빈도 차이가 매우 큼.
2. **정상 이미지의 높은 비율**: 전체 이미지의 약 60.4%가 질환이 없는 정상 이미지로 구성됨.
3. **고밀도 다중 병변**: 단일 이미지에 최대 57개의 질환 바운딩 박스가 존재함.
4. **다양한 병변 크기**: 미세한 결절부터 넓은 기흉까지 병변 크기의 스펙트럼이 넓음.

## 3. 해결 전략 및 파이프라인 (Methodology)

### 3.1. 통합 전처리 및 포맷팅
- **배경(Background) 이미지 처리**: 60%에 달하는 정상 이미지를 학습에서 제외하지 않고, 빈 라벨 텍스트 파일을 생성하여 모델이 정상 패턴(배경)을 학습하고 오탐(FP)을 줄일 수 있도록 유도했습니다.
- **유연한 확장자 처리**: 다중 포맷의 의료 이미지에 대응하기 위해 재귀적 탐색 기반의 동적 확장자 매칭 로직을 구현했습니다.

### 3.2. 모델 설계 (YOLOv8m)
다중 스케일 탐지에 유리하며 Kaggle 환경(15GB VRAM, 12시간 런타임 제한)에 적합한 YOLOv8m(Medium) 아키텍처를 채택했습니다.
- **하이퍼파라미터 최적화**: 밀집 병변 탐지를 위해 탐지 상한치를 상향(`max_det=300`)하고, VRAM을 최대한 활용하기 위해 배치 사이즈를 조절(`batch=28`)했습니다. 
- **의료 데이터 맞춤형 증강**: 클래스 불균형 완화를 위해 모자이크 및 믹스업 증강을 적용하되, 방사선 사진의 해부학적 특성을 고려해 상하 반전과 색상 왜곡 등의 증강은 제거하여 훈련 안정성을 높였습니다.

### 3.3. 엔지니어링 및 트러블슈팅
- **DDP(분산 학습) 충돌 해결**: 다중 GPU 학습 시 정상(Background-only) 배치에서 발생하던 DFL 파라미터 Gradient 연산 오류를 해결하기 위해, 런타임에 소스 코드를 수정하여 `find_unused_parameters=True`를 주입하는 패치를 적용했습니다.
- **OOM 및 호환성 방어**: 파이토치 메모리 단편화를 방지하기 위해 `expandable_segments` 환경 변수를 설정하고, 실행 환경의 GPU 아키텍처(T4x2)에 맞춰 적합한 PyTorch 버전이 동적으로 설치되도록 스크립트를 구성했습니다.

### 3.4. 평가 및 추론 최적화
- **사용자 정의 mAP@0.4 평가**: YOLO 프레임워크가 기본적으로 COCO 기준(IoU 0.5)으로 평가를 진행하는 점을 보완하기 위해, `torchmetrics`를 활용하여 대회의 공식 지표인 mAP@0.4를 훈련 종료 후 계산하는 커스텀 평가 루프를 구현했습니다.
- **임계값 튜닝 및 원본 해상도 스케일링**: Kaggle 평가 시스템의 PR Curve 컷오프를 방지하기 위해 추론 신뢰도 임계값을 `0.01`로 하향 조정했습니다. 또한, 이미지 내 예측된 최대 신뢰도가 `0.15` 미만인 경우에는 강제로 `No finding` 클래스를 할당하여 정상 이미지에 대한 정확도를 방어했습니다.
